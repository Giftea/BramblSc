With a wallet, you can generate key pairs for use in the blockchain network. Our implementation of a 
hierarchical deterministic (HD) wallet uses [Extended Ed25519](https://pub.dev/documentation/bip32_ed25519/latest/). 
**HD wallets allow you to derive multiple child key pairs from a single main key, which helps in better managing 
cryptocurrencies by separating transaction keys from master keys.** 

This page will walk you through the process of accessing your main key pair and deriving child key pairs for a 
3-layer path (x/y/z).

## Prerequisites

To use a wallet, a wallet instance must exist. To create a wallet instance, read [Create a Wallet](./create) or [Import a Wallet](./import).

:::note
This implementation uses [Cats Effect](https://typelevel.org/cats-effect/docs/2.x/datatypes/io), a library for functional programming in Scala, which provides the capability to work with asynchronous and concurrent programming. 
:::

The following snippet demonstrates how to create a new wallet instance using Cats Effect `IO` and the default implementation of the <ScaladocLink path="co/topl/brambl/servicekit/index.html"><code>ServiceKit</code></ScaladocLink>
:

```scala
import cats.effect.IO
import co.topl.brambl.wallet.WalletApi
import co.topl.brambl.servicekit.WalletKeyApi

val password = "password".getBytes
val walletApi = WalletApi.make[IO](WalletKeyApi.make())
val newWalletResult = walletApi.createNewWallet(password)
```

## Decrypt a Main Key Pair

During wallet initialization, a main key pair is generated and encrypted in a `VaultStore`. To use this key pair
you must first decrypt it by extracting it from the `VaultStore`. This can be achieved by using
the <ScaladocLink path="co/topl/brambl/wallet/WalletApi.html#extractMainKey(VaultStore[F],Array[Byte]):F[Either[WalletApiFailure,KeyPair]]"><code>extractMainKey</code></ScaladocLink>
function of a Wallet API instance.

```scala
def extractMainKey(
  vaultStore: VaultStore[F],
  password:   Array[Byte]
): F[Either[WalletApi.WalletApiFailure, KeyPair]]
```

This function extracts and decrypts the main key pair from the given encrypted `VaultStore` using the provided password.

**Parameters:**

- `vaultStore`: The encrypted `VaultStore` containing the main key pair.
- `password`: The password used to encrypt the `VaultStore`.

On success, the function returns the decrypted
main [key pair](https://github.com/Topl/protobuf-specs/blob/main/proto/quivr/models/shared.proto#L78). On failure, the function returns
a <ScaladocLink path="co/topl/brambl/wallet/WalletApi$$WalletApiFailure.html"><code>WalletApiFailure</code></ScaladocLink>
which specifies the reason for failure.

:::caution
For maximum security, the main key pair should never be used to create and prove transactions. Instead, always use a child key pair derived from the main key pair.
:::

### Example

The following example shows how to extract the main key pair after initializing a wallet.

```scala
import cats.effect.IO
import cats.effect.unsafe.implicits.global
import co.topl.brambl.wallet.WalletApi
import co.topl.brambl.servicekit.WalletKeyApi

val password = "password".getBytes
val walletApi = WalletApi.make[IO](WalletKeyApi.make())

val mainKey = for {
  newWallet <- walletApi.createNewWallet(password)
  // highlight-next-line
  decryptedKey <- walletApi.extractMainKey(newWallet.toOption.get.mainKeyVaultStore, password)
} yield decryptedKey.toOption.get

mainKey.unsafeRunSync()
```

## Load and Decrypt a Main Key Pair

If your wallet was saved to the Wallet Key persistence layer during wallet initialization (for example, by
using [Create and Save a Wallet](./create#create-and-save-a-wallet) or [Import and Save a Wallet](./import#import-and-save-a-wallet)),
your main key pair can be loaded and decrypted in a single step. This can be achieved by using
the <ScaladocLink path="co/topl/brambl/wallet/WalletApi.html#loadAndExtractMainKey[G[_]](Array[Byte],String)(Monad[G],ToMonad[G]):G[Either[WalletApiFailure,KeyPair]]"><code>loadAndExtractMainKey</code></ScaladocLink>
function of a Wallet API instance.

```scala
def loadAndExtractMainKey[G[_]: Monad: FunctionK[F, G]](
  password: Array[Byte],
  name:     String = "default"
): G[Either[WalletApi.WalletApiFailure, KeyPair]]
```

This function loads an encrypted wallet (specified by `name`) from the `WalletKeyApiAlgebra` used to initialize the Wallet API, and then extracts and decrypts the main key pair contained within it using the provided password.

**Parameters:**

- `password`: The password used to encrypt the `VaultStore`.
- `name`: The name of the wallet to load. Defaults to "default".

**Type Parameters:**

- `G`: The context, bound to a context parameter of type `Monad[G]` and transformable to `F` (using Cats `FunctionK`), where `F` is the type parameter used in the WalletApi instance. Most commonly, `G` will be the same as `F` and will be transformed using `FunctionK.id[F]`.

On success, the function returns the decrypted
main [key pair](https://github.com/Topl/protobuf-specs/blob/main/proto/quivr/models/shared.proto#L78). On failure, the function returns
a <ScaladocLink path="co/topl/brambl/wallet/WalletApi$$WalletApiFailure.html"><code>WalletApiFailure</code></ScaladocLink>
which specifies the reason for failure.

:::caution
For maximum security, the main key pair should never be used to create and prove transactions. Instead, always use a child key pair derived from the main key pair.
:::

### Example

The following example shows how to load a wallet and extract its main key pair after previously persisting it using
the default implementation of the `WalletKeyApi` provided by
the <ScaladocLink path="co/topl/brambl/servicekit/index.html"><code>ServiceKit</code></ScaladocLink>. This default implementation
of the `WalletKeyApi` uses the local filesystem to persist the wallet and mnemonic.

```scala
import cats.arrow.FunctionK
import cats.effect.IO
import cats.effect.unsafe.implicits.global
import co.topl.brambl.wallet.WalletApi
import co.topl.brambl.servicekit.WalletKeyApi

import java.io.File

val walletApi = WalletApi.make[IO](WalletKeyApi.make())
implicit val transformType: FunctionK[IO, IO] = FunctionK.id[IO]

val homeDir = System.getProperty("user.home")
// Replace with the desired location for your key file
val keyFile = new File(homeDir, "mainkey.json").getCanonicalPath
// Replace with the desired location of your mnemonic file
val mnemonicFile = new File(homeDir, "mnemonic.txt").getCanonicalPath

val password = "password".getBytes

walletApi.createAndSaveNewWallet[IO](password, name = keyFile, mnemonicName = mnemonicFile).unsafeRunSync()

// Load and extract begins here:
val mainKey = for {
  // highlight-next-line
  decryptedKey <- walletApi.loadAndExtractMainKey[IO](password, keyFile)
} yield decryptedKey.toOption.get
mainKey.unsafeRunSync()
```

## Derive a Child Key Pair

The main key pair should not be used to create and prove transactions. For this, a child key pair should be used.
You can derive this child key pair using
the <ScaladocLink path="co/topl/brambl/wallet/WalletApi.html#deriveChildKeys(KeyPair,Indices):F[KeyPair]"><code>deriveChildKeys</code></ScaladocLink>
function of a Wallet API instance.

```scala
def deriveChildKeys(
  keyPair: KeyPair,
  idx:     Indices
): F[KeyPair]
```

This function derives a child key pair from the given main key pair using the specified indices. The indices contain the 3-layer path (x/y/z) for the child key pair. For a more formal definition of the derivation paths, see [TIP-0004](https://github.com/Topl/tips/tree/main/TIP-0004). For the most basic use case, the `x/y` path can be hardcoded to a constant value (e.g., `10/10`), and the `z` path can be incremented for each new child key pair needed.

**Parameters:**

- `keyPair`: The main key pair from which to derive the child key pair.
- `idx`: The indices containing the 3-layer path (x/y/z) for the child key pair.

This function returns the derived child [key pair](https://github.com/Topl/protobuf-specs/blob/main/proto/quivr/models/shared.proto#L78).

### Example

The following example shows how to derive 2 different child key pairs from the main key pair.

```scala
import cats.effect.IO
import cats.effect.unsafe.implicits.global
import co.topl.brambl.models.Indices
import co.topl.brambl.wallet.WalletApi
import co.topl.brambl.servicekit.WalletKeyApi

val password = "password".getBytes
val walletApi = WalletApi.make[IO](WalletKeyApi.make())

val mainKey = for {
  newWallet <- walletApi.createNewWallet(password)
  decryptedKey <- walletApi.extractMainKey(newWallet.toOption.get.mainKeyVaultStore, password)
} yield decryptedKey.toOption.get

// highlight-next-line
val childKey1 = mainKey.map(key => walletApi.deriveChildKeys(key, Indices(10, 10, 1)))
val childKey2 = mainKey.map(key => walletApi.deriveChildKeys(key, Indices(10, 10, 2)))

childKey1.unsafeRunSync()
childKey2.unsafeRunSync()
```

### Path Indices Explanation

In the example above, the indices are specified as `Indices(10, 10, z)`, where `z` is incremented for each new child key pair derived. The first two indices (10/10) represent a hardcoded path, while the third index varies with each call. This allows for multiple transactions to use distinct keys without compromising the main key pair's security.

:::

## Conclusion

Now that you have learned how to extract and derive keys, consider exploring how to use the derived child key pairs in transactions to enhance the security and management of your cryptocurrency.
